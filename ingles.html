what is HTML?

HTML is the language with which the content of web pages is defined. Basically it is a set of labels that serve to define the text and other elements that will make up a web page, such as images, lists, videos, etc. HTML was originally created for informational informational purposes with text and some images. It was not thought that it would come to be used to create a multimedia leisure and consultation area (which is currently the web), so that HTML was created without responding to all the possible uses that were going to be given to it already. all groups of people who would use it in the future.

2. Headers

HTML

HTML provides not just simple paragraph tags, but six separate heading tags to indicate headings of various sizes and thicknesses. Listed as heading 1 to heading 6, heading 1 has the largest and thickest text, heading 6 is the smallest and thinnest, down to paragraph level. This topic details the proper use of these tags.

2.1 Use of titles

HTML

Headings can be used to describe the topic they precede and are defined with the < h1> to < h6> tags. Headers supports all global attributes. < h1> defines the most important heading. < h6> defines the least important heading. Definition of a header: < h1> Heading 1 < / h1>
< h2> Heading 2 < / h2>
< h3> Heading 3 < / h3>
< h4> Title 4 < / h4>
< h5> Heading 5 < / h5>
< h6> Heading 6 < / h6>

2.2 correct structure matters

HTML

Search engines and other user agents often index page content based on header elements, for example to create a table of contents, so it is important to use the correct structure for headings. In general, an article should have an h1 element for the main title followed by h2 subtitles, lowering a layer if necessary. If there are h1 elements at a higher level, they should not be used to describe any lower-level content. Sample document (additional intent to illustrate the hierarchy): < h1> Main title < / h1>
< p> Introduction < / p>
< h2> Reasons < / h2>
< h3> Reason 1 < / h3>
< p> Paragraph < / p>
< h3> Reason 2 < / h3>
< p> Paragraph < / p>
< h2> In conclusion
< p> Paragraph < / p>

4. Grids

CSS

Grid layout is a powerful new CSS layout system that allows you to divide the content of a web page into rows and columns in an easy way.
4.1: Basic example
Property Possible values display grid / online grid The CSS grid is defined as a display property. Applies to a parent element and its immediate children only. Consider the following markup: < section class = "container">
< div class = "item1"> item1 < / div>
< div class = "item2"> item2 < / div>
< div class = "item3"> item3 < / div>
< div class = "item4"> item4 < / div>
The easiest way to define the above markup structure as a grid is to simply set its display property to grid: .package {
screen: grid;
}
However, doing this will always cause all the children to collapse on top of each other. This is because children currently do not know how to position themselves within the grid. But we can tell you explicitly. First we need to tell the grid element .container how many rows and columns will make up its structure and You can do this using the properties of the grid columns and the grid rows (note pluralization): .package {
screen: grid;
grid columns: 50px 50px 50px;
grid rows: 50px 50px;
}
However, that still doesn't help us much because we need to give an order to each child element. We can do this for specifying the grid row and grid column values ​​that will tell you where you are on the grid: .container .item1 {
grid column: 1;
grid row: 1;
}
.container .item2 {
grid column: 2;
grid row: 1;
}
.container .item3 {
grid column: 1;
grid row: 2;
} .container .item4 {
grid column: 2;

grid row: 2;
}
By giving each item a column and row value, you identify the order of the items within the container. See a working example in JSFiddle. The overall height of the menu will increase as needed.
